# 1.2.3 Booleans - Introduction

## Quick concept map

- Boolean values: True and False (and None for an empty value).
- Comparison blocks compare values and produce booleans.
- Logical blocks combine boolean results with `and` / `or`.
- Unary blocks negate or check identity (`not`, `is None`, `is not None`).
- The conditional (ternary) block chooses between two values depending on a condition.

## 1) Comparison block (pill with dropdown)

Python equivalents and examples:

```python

# comparisons

```

## 2) Conditional expression block (ternary)

<img
  src="/1-fundamentals-of-programming/1.2-data-types-operations/1.2.3-booleans/res/BOO-1.3.png"
  alt="Ternary conditional block: value_if_true if condition else value_if_false"
  style={{ width: "50%" }}
/>

This block maps directly to Python's `x if cond else y` expression. It's handy for short choices.

```python
print(1 if True else 0)         # prints 1
print(100 if 3 < 2 else -100)   # prints -100

# nested ternary (read carefully):
print(100 if True else (-100 if True else 0))
```

Tip: keep nested ternaries short; prefer normal `if` statements for complex logic.

## 3) Logical operator block (`and` / `or`)

<img
  src="/1-fundamentals-of-programming/1.2-data-types-operations/1.2.3-booleans/res/BOO-1.4.png"
  alt="Logical operator block: left and/or right"
  style={{ width: "50%" }}
/>

Python examples:

```python
print(True and False)   # False
print(True or False)    # True

# combine comparisons
print((3 < 4) and (5 > 2))  # True
```

Small note on short-circuiting: `and` and `or` return the first value that decides the result in Python â€” this matters when their operands are non-boolean values.

## 4) Unary boolean block (`not`, `is none`, `is not none`)

<img
  src="/1-fundamentals-of-programming/1.2-data-types-operations/1.2.3-booleans/res/BOO-1.5.png"
  alt="Unary boolean block: not / is none / is not none"
  style={{ width: "50%" }}
/>

Examples:

```python
print(not True)        # False

value = None
print(value is None)   # True
print(value is not None) # False
```

`is None` checks identity against `None` (recommended for checking emptiness).
